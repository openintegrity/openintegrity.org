1. Contributor

To add a report about a threat or incident, follow the steps below.

  1. If you do not yet have one, create an account on the iilab gitlab
  2. Confirm your email address
  3. Sign in
  4. Go to the reports folder of the impact repository
  5. Click the [+] button and select New file

  6. Enter a filename in the File Name field according to the format below

    yyyy-mm-dd-<description>

  7. Add front matter (meta-data) to the report according to the format below

    ---
reference: <URL if relevant>
reference_type: <report, news, primary, etc.>
source: <organisational URL, primary>
source_type: <academia, media, corporate, individual, etc.>
tool: <tool to which the report is relevant>
community: <community affected>
---

  8. Select an appropriate impact pattern from the list here and add it to the front matter according to the format below. Do not include the .md extension.

    ---
pattern: <pattern>
---
  9. If you are reporting an original incident, describe it in the body of the report, below the front matter. Othewise, leave the body of the report blank. Be sure not to reveal sensitive or identifying information about the individuals involved.
  10. Type a brief but meaningful Commit message along the lines of "Added a report about <description>"
  11. Click [Commit changes]. The framework will let you know that a new branch will be created in your fork and a new merge request will be started.
  12. Use the Description box to provide background or clarifying information to help us understand your report. Examples might include:
    • Which section(s) of a linked document are relevant
    • Concerns about how well this report fits with the chosen impact pattern
    • <???>

  13. Click [Submit merge request]. The framework will let you know that your report has been submitted and send you to a page where you can keep track of ongoing discussion about this report. Please make note of the URL, though we will likely contact you by email if we need additional information.
