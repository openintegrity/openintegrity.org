1. Contributor

How to add and tag an incident

  1. Create an account on the iilab gitlab
  2. Confirm your email address
  3. Sign in
  4. Go to the reports folder
  5. Click [+] and select New file.

  6. Click File Name and enter a filename as shown below

    yyyy-mm-dd-<description>

  7. Add the report meta-data as shown below

    ---
reference: <URL if relevant>
reference_type: <report, news, primary, etc.>
source: <organisational URL, primary>
source_type: <academia, media, corporate, individual, etc.>
tool: <tool to which the report is relevant>
community: <community affected>
...

  8. Select an impact pattern from the list here and add it to the front matter

    ...
pattern: <pattern>
---
  9. If you are reporting an original incident, describe it anonymously in the body of the report. Be do not reveal sensitive or identifying information about the individuals involved.
  10. Type a simple but descriptive Commit message along the lines of "Added a report about "
  11. Click [Commit changes]. A new branch will be created in your fork and a new merge request will be started.
  12. Use the Description box to provide background or clarifying information to help us understand your report

  13. Click [Submit merge request]

Your report has been submitted. Make note of the URL, as further discussion about this report will be visible here. We will contact you by email if we need additional information.
