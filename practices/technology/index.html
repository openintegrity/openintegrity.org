Technology Practice

How a tool is developed, compiled, distributed and maintained has a clear impact on end user privacy and security. As does the architecture and administration of any back-end infrastructure upon which it might depend.

The following practices realted to the technology aspect of software development are linked to end user privacy and security.

Dependency management

Architecture

Vulnerabilities

Distribution of Software

  • Distributes software through signed packages
  • Provides binary for HTTPS download with HSTS
  • Provides binary for HTTPS download
  • Hosts checksums for binary through HTTPS
  • Makes binary available in repository or app store, signed by repository administrator
  • Makes binary available in repository or app store, signed by developer
  • Provides a binary installer (.msi, .deb, etc.)
  • Provides a lightweight network installer
  • Provides a binary executable blob (.bin, .exe, etc.)
  • Makes binary available through a free file host (Sourceforge, tucows, etc.)
  • Makes binary available through HTTP download

Development Team

  • Enforces strong "hygiene" practices on developers' machines
  • Provides developer documentation (specs, threat model, etc.)
  • Produces reproducible builds
  • Uses dedicated machines for development
  • Uses automation tools for scanning
  • Uses automation tools for build & release
  • Has a key- and access-revocation process
  • Lacks policies on device compartmentalization (build machines vs. dev machines vs. personal machines, etc.)
  • Uses development machines for various purposes
  • Distributes malware in the installation package or binary blog
  • Inserts malicious code into the product before build
  • Builds software on development machines inflicted with malware
