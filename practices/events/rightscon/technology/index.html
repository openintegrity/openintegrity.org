Technology Practice

The Technology practices identified at the metrics workshop are listed below. Follow links for details.

Distribution of Software

Good practice

  • Signed packages
  • HTTPS download with HSTS
  • Binary checksums
  • HTTPS download
  • App store with repo signature
  • App store with developer signature

OK practice

  • Installer download (.msi, .deb, etc.)
  • Lightweight network installer
  • Binary executable blob (.bin, .exe, etc.)

Poor practice

  • Free file host (Sourceforge, tucows, etc.)
  • HTTP download

Development Team

Good practice

  • Use software hygiene practices on developers' machines
  • Developer documentation
  • Reproducible builds
  • Use dedicated machines for development
  • Automation tools
  • Key and access revocation process

Poor practice

  • Non-existent policies regarding machine use for other purposes
  • Use of development machines for many purposes
  • Malicious software included in the installation package
  • Malicious coce inserted in the product before build
  • Malware injected in development machines
