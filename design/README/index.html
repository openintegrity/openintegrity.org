Open Integrity Design Repo

This repo holds ideas and designs for the Open Integrity Initiative's user experience.

Personas

The main audience for the Open Integrity website is an audience of professionals. Here are a few examples of personas that would be potential users of the site:

  • Users
    • Tool Developers (including internet freedom tools, mainstream commercial tools): Interested in making sure their tool is properly represented, contribute to the discussions on best practices, learn more about how to adopt best practices. They might also help ensure that information about other tools is correct.
    • Trainers (including digital security trainers): Interested in finding the latest about specific tools.
    • Advocates (such as EFF, Access Now...): Interested in having the latest evidence about tools in order to support awareness raising or other campaigns. Also interested in creating a Score which will assemble metrics in a comparable fashion.
  • Contributors
    • Reviewers (experts in legal, technical, usability,...): Interested in ensuring that metrics are correct and fair.
  • Partners
    • Measurement Partners (such as Libraries.io / OONI / Google Open Source / Simply Secure / Second Muse / Ranking Digital Rights / TOSDR; / Privacy International / iSec Partners / Cure 53 / Core Infrastructure Initiative): Interested in disseminating their measurement or research data through Open Integrity.

User Stories

  • As a user, I can search for a specific software package or service to view metrics on it
  • As a user, I can view all available metrics for a software package or service
  • As a user, I can become a contributor by submitting a measurement/observation.
  • As a measurement partner, I can provide metrics to OII
